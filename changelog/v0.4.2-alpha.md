# StreamSuites-Public v0.4.2-alpha

`StreamSuites-Public v0.4.2-alpha` tightens the public shell around auth-gate UX, profile and share-link polish, and cleaner release-facing public surfaces. The work in this cycle keeps the repo downstream of runtime and Auth API authority while making the public experience more stable and more coherent.

## 🔐 Public Auth Gate and Banner UX

- Wired public auth entry points and login surfaces to the runtime-owned `/auth/access-state` and `/auth/debug/unlock` flow instead of inventing local lockout rules.
- Added the passive lockout banner behavior so maintenance or development mode can be communicated without turning ordinary public pages into gated routes.
- Followed with redirect-loop, category-hydration, and auth-modal fixes so public login and browse paths remain usable during gate-state changes.

## 👤 Public Shell, Profiles, and Share Behavior

- Tightened top-bar user widget alignment, fallback avatar behavior, footer polish, and profile share-link handling across the public shell.
- Continued the public profile work around canonical slug routing, truthful dual-share behavior, and FindMeHere-aware visibility and eligibility presentation.
- Kept live badges, live rings, and related profile treatments tied to centralized runtime live-status state instead of repo-local authority.

## 🧰 Public Surface and Packaging Polish

- Added and refined public-facing resources and statistics surfaces as release-grade companion pages.
- Refreshed sample clip media and packaging so demo and gallery content better match the current public release state.
- Kept the repo’s hydrated changelog JSON in place for the website while adding this Markdown file as the release-package source artifact.
